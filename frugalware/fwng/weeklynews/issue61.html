Events etc

Here's a selection of events which have occurred since the previous newsletter:

Does your favourite editor support FrugalBuild files?

Frugalware developer ryuo is working to enable syntax highlighting for FrugalBuild files in all text editors packaged in Frugalware. If your favourite editor doesn't already have this feature, please post a feature request in the Frugalware Bug Tracker System.

GNOME 2.30 is available in 'current'

The GNOME team recently announced the release of the latest in the 2.x series - 2.30. Soon after, Frugalware developer bouleetbil announced that he had started packaging it. GNOME is a complex collection of packages, so it takes a lot of effort to upgrade all the FrugalBuilds required. Bouleetbil made it look easy, however, and it's now available in the 'current' repository.

Turkish translation has started

Tutku "Titus" DALMAZ has started the task of translating the various Frugalware projects into Turkish. This is great news for Frugalware because it provides easier access to Turkish speakers. More translations are always welcome because one of Frugalware's earliest goals was to provide multi-lingual distribution. Thankyou, Titus for starting this work and we hope to hear about your progress soon.

The battle of the game packagers has begun!

Adrien Ramos recently submitted a new FrugalBuild for a game and in the commit comment was "New package : a game, I want to dethrone devil505 for the games! Ahahah!". This is the first time anyone has challenged another packager, so it was an interesting move. Devil505 has packaged many games though, so Adrien has a lot of work to do. Let's hope that there's not too much bloodshed in the battle to come. :P

GNOME is asking for money

The GNOME foundation recently started a campaign, asking people and companies for money to employ a sysadmin "so that GNOME developers can focus on GNOME." Their goal is to raise $32,000 and with the help of many companies and individual contributors, they have almost reached this goal. If you use GNOME, please consider making a contribution.

Tips and tricks

Disclaimer - Be aware that the hints & tips provided here have NOT been tested and so come with no warranty.

Tell me when you're finished

When you're working in a CLI you sometimes have to wait for something to finish. It might be a file download, or burning an image to a CD or DVD. Instead of staring at the screen while waiting for the task to finish, you can have your computer tell you when it's complete.

You can have a favourite audio clip played when the command is finished, simply by adding "&& play audio_clip" at the end of the command. This requires the sox package installed. You might prefer a popup message on the screen and to do this simply add "&& notify-send message -t 0" at the end of the command. The "-t 0" parameter means that the popup message will remain displayed until it's clicked on. This requires the libnotify package installed and popup notifications enabled. The "&&" parameter tells bash that the command which follows it is to be run only when the first command is complete.

As an example, you want an audio clip played when a wget command is complete:

wget http://www.mirror.com/frugalware.iso && play ~/audio/complete.ogg

In another example, you want a pop-up notification when a "git pull" is complete:

git pull && notify-send "git pull is complete" -t 0

These are just examples of ways in which you can get on with other tasks while a long-running task completes in the background. A simple search of the Internet will reveal many more ways of achieving the same goal, with some being more complex than others.

Focus On Package(s)

Pino

Pino is a microblogging client which is compatible with Twitter and, more importantly, Status.net sites such as identi.ca. It has a clear, easy user interface but it has a great range of features without being bloated. It's still under heavy development, with additional features being added with each release and bugs being hunted down and squashed. If you use either the Twitter or Status.net services, please try Pino, even if you currently use another client.
