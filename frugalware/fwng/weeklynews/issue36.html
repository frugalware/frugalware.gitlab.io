In this section of the newsletter, we get to know Frugalware Linux a little at a time. This is not meant to replace the official documentation but instead make it easier to read. This should suit those those people who, like me, have short attention spans. :)

What's in a name?

Frugalware's package naming convention is quite simple but it's different from the one used by other distributions, including Ubuntu. The naming convention has 3 parts, in the following order:

  1. application/library name (sourced from upstream, unless it conflicts with the name of an existing package)
  2. application/library version number (sourced from upstream)
  3. Frugalware package revision number

Let's look at the name of the CUPS package, for example. It's named cups-1.3.9-1, so this is revision 1 of the cups package, upstream version 1.3.9. If a package is upgraded upstream, the Frugalware revision number starts again from 1.

Note: anyone who thinks that the revision number should start from 0 probably needs to spend a little more time away from their PC. ;P

Tips and tricks

 Disclaimer - Be aware that the hints & tips provided here have NOT been tested and so come with no warranty.

  • SciTE - submitted by phayz

    SciTE - a text editor - displays its in-built toolbar icons by default and they're not very good looking. If you add the option "toolbar.usestockicons=1" to the user's configuration file, the toolbar will use your GTK theme's icons. This looks a lot better than the default because it makes SciTE more like a native application.

  • How much disk space is free? - submitted by phayz

    Like all operating systems, Linux needs disk space to work. Therefore it's a good idea to regularly check how much disk space is free on all partitions. The simplest way to do this is with the command "df -h" in a console. This will list each filesystem and the amount of free disk space on it. The "-h" switch tells the DF command to display the disk space figures in human-readable format, not in bytes.

  • artwiz fonts

    The artwiz fonts is a small collection of bitmap fonts. I'm not sure if they're still actively being developed but there hasn't been much activity at their home page for a long time. Most of them are quite small but their clarity is excellent. Personally I use the "snap" font everywhere on my desktop except for window titles. It's very easy to read even though it's only about 8 point in size. I use it because it makes most dialog boxes and menus smaller than they would be if I used a font such as DejaVu Sans. I run X at a resolution of 1280x1024 on a 17 inch LCD monitor and I can read the text easily. For those running higher resolutions and/or larger monitors, it might be too small to read. Anyway, please try them and you might find that they suit you perfectly.

Focus On Package(s)

SLIM

SLIM is a login manager and its name is formed from the words "Simple Login Manager". It's described as "simple" because it's not dependent upon any desktop environment and so is light on resource usage. It's very much like GDM or KDM in terms of functionality. At the login screen you can select with the [F1] key which desktop environment or window manager to start. SLIM comes with some basic themes but you can make your own if you don't like any of those. It's perhaps best suited to those who are running a window manager because GDM and KDM offer more options. If you choose to try SLIM, you'll need to edit /etc/sysconfig/desktop and uncomment the line containing "slim" and comment all other lines. When you next reboot, SLIM will appear immediately after boot.
