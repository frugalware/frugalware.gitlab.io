Welcome

You may have noticed that it's been some time since the previous newsletter (25 July, 2007 to be exact). AlexExtreme - the previous author of the newsletter - has moved on to other projects. Many thanks go to AlexExtreme for starting the newsletter. I (Phayz) have taken on the task of publishing the newsletter in his place and intend carrying on his good work. My aim is to return to a two-week release as quickly as possible.

Events

Here's a selection of events which occurred since the previous newsletter:

  • Farewell to AlexExtreme

    On 12 September, AlexExtreme (Alex) announced that he was leaving the Frugalware development team. His other projects, school work etc meant that he was forced to choose between them. Frugalware's loss is (hopefully) a gain to his education. Alex's areas of contribution to Frugalware were as a package maintainer, Gnome co-maintainer and web site/wiki maintainer. He and his contributions to the development of Frugalware will be missed. Good luck to him for the future.

  • Welcome to Devil505

    On 28 October, the Frugalware developers voted that Devil505 (Daniel) be welcomed to the development team. He'd recently contributed several packages for software new to the Frugalware repository. Welcome to Daniel and we look forward to many more packages in the future.

  • Frugalware users have more FUN!!

    On 22 November, Priyank Gosalia announced the release to the current repository of the Frugalware Update Notifier (FUN) version 0.1.0. FUN is a update notifier that sits in your system tray checking for new package updates at regular intervals.

Tips and tricks

  • KDE - configuring the clock - submitted by 'phinger'

    You may find that all the clock's configuration options are greyed out. To fix this, remove the clock from the panel, then add it. When you again choose the 'Configure Clock' menu option, you should find all the configuration options are available.

This section relies on your contributions! If you have some tips and tricks that you would like to be shown in the newsletter, please post them on the forums in the Tips and Tricks section!

Security announcements

According to the normal support arrangements for Frugalware, when Frugalware 0.7 (Sayshell) was released, support for the previous release ended. This means that no further security or bug fixes will be released.

Here is a list of security issues which have been discovered and fixed in the 0.7 release since its release.

  • FSA319 - poppler: Some vulnerabilities have been reported in Poppler, which can be exploited by malicious people to compromise an application using the library. Please upgrade to poppler-0.6-2sayshell1.
  • FSA318 - kdegraphics: Some vulnerabilities have been reported in KOffice, which can be exploited by malicious people to compromise a user's system. Please upgrade to kdegraphics-3.5.7-3sayshell1.
  • FSA317 - koffice: Some vulnerabilities have been reported in KOffice, which can be exploited by malicious people to compromise a user's system. Please upgrade to koffice-1.6.3-3sayshell1.
  • FSA316 - xpdf: Secunia Research has discovered some vulnerabilities in Xpdf, which can be exploited by malicious people to compromise a user's system. Please upgrade to xpdf-3.02-4sayshell1.
  • FSA315 - kernel: A vulnerability has been reported in the Linux Kernel, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to kernel-2.6.22-7sayshell2.
  • FSA314 - liferea: A security issue has been reported in Liferea, which can be exploited by malicious, local users to disclose sensitive information. Please upgrade to liferea-1.2.23-2sayshell1.
  • FSA313 - django: A vulnerability has been reported in Django, which potentially can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to django-0.96-2sayshell1.
  • FSA312 - cups: Secunia Research has discovered a vulnerability in CUPS, which can be exploited by malicious people to compromise a vulnerable system. Please upgrade to cups-1.3.2-2sayshell1.
  • FSA311 - wordpress: Janek Vind has discovered a vulnerability in WordPress, which can be exploited by malicious people to conduct cross-site scripting attacks. Please upgrade to wordpress-2.3.1-1sayshell1.
  • FSA310 - kernel: Evan Teran has reported a security issue in the Linux kernel, which potentially can be exploited by malicious, local users to cause a DoS (Denial of Service). Please upgrade to kernel-2.6.22-7sayshell1.
  • FSA309 - asterisk-addons: A vulnerability has been reported in Asterisk-Addons, which can be exploited by malicious people to conduct SQL injection attacks. Please upgrade to asterisk-addons-1.4.4-1sayshell1.
  • FSA308 - asterisk: A vulnerability has been reported in Asterisk, which can be exploited by malicious people to cause a DoS (Denial of Service) or potentially to compromise a vulnerable system. Please upgrade to asterisk-1.4.13-1sayshell1.
  • FSA307 - tomboy: Jab Oravec has reported a security issue in Tomboy, which can be exploited by malicious, local users to gain escalated privileges. Please upgrade to tomboy-0.8.0-2sayshell1.
  • FSA306 - seamonkey: Some vulnerabilities and a weakness have been reported in Mozilla SeaMonkey, which can be exploited by malicious people to disclose sensitive information, conduct phishing attacks, manipulate certain data, and potentially compromise a user's system. Please upgrade to seamonkey-1.1.5-1sayshell1.
  • FSA305 - firefox: Some vulnerabilities and a weakness have been reported in Mozilla Firefox, which can be exploited by malicious people to disclose sensitive information, conduct phishing attacks, manipulate certain data, and potentially compromise a user's system. Please upgrade to firefox-2.0.0.8-1sayshell1.
  • FSA304 - drupal: Some vulnerabilities have been reported in Drupal, which can be exploited by malicious people to conduct cross-site scripting attacks and bypass certain security restrictions, and by malicious users to conduct HTTP response splitting attacks. Please upgrade to drupal-5.2-2sayshell1.
  • FSA303 - opera: Some vulnerabilities have been reported in Opera, where one vulnerability has an unknown impact and others can be exploited by malicious people to conduct cross-site scripting attacks and to compromise a user's system. Please upgrade to opera-9.24-1sayshell1.
  • FSA302 - hplip: Kees Cook has reported a vulnerability in HPLIP, which can be exploited by malicious, local users to gain escalated privileges. Please upgrade to hplip-2.7.7-2sayshell1.
  • FSA301 - madwifi: Clemens Kolbitsch and Sylvester Keil have reported a vulnerability in MadWifi, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to madwifi-0.9.3.3-1sayshell1.
  • FSA300 - phpmyadmin: Omer Singer has reported two vulnerabilities in phpMyAdmin, which can be exploited by malicious people to conduct cross-site scripting attacks. Please upgrade to phpmyadmin-2.11.1.2-1sayshell1.
  • FSA299 - openssl: Andy Polyakov has reported a vulnerability in OpenSSL, which can be exploited by malicious people to cause a DoS (Denial of Service) and potentially compromise a vulnerable system. Please upgrade to openssl-0.9.8-10sayshell1.
  • FSA298 - pwlib: A vulnerability has been discovered in PWLib, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to pwlib-1.10.10-2sayshell1.
  • FSA297 - wesnoth: A vulnerability has been reported in Wesnoth, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to wesnoth-1.2.6-2terminus1.
  • FSA296 - libpng: Some vulnerabilities have been reported in libpng, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to libpng-1.2.22-1sayshell1.
