News

We developers have been doing some cool stuff with Frugalware over the last 2 weeks. Here's what's been happening:

  • XULRunner

    A new XULRunner package has been added in -current this week and all GNOME apps that have depended upon Firefox for HTML rendering have been rebuilt against XULRunner. This has been requested for ages, and now that it has been done you no longer have to install Firefox if you just want to use Epiphany or Galeon.

  • FwLive

    We have made a lot of progress with FwLive, our new LiveCD, over the past few weeks. You can see about the progress and get the latest snapshot ISO image here.

  • New Frugalware 0.7 GTK+2 theme

    A new GTK+2 theme was uploaded to -current last week, and it is now the default theme for GNOME and XFCE. There are a few known problems with it, such as the nasty looking tabs, but they will be fixed in due course. You can see a screenshot here.

  • Pidgin in -current

    Many of you who use Gaim will have known that it has been renamed to Pidgin after a legal problem with AOL. Pidgin 2.0.0 was released a few days ago, it has been added in -current and all the plugins have been updated accordingly. This new version comes with many features and brand new icons. I'd like to congratulate the Pidgin developers for a great release :)

  • OpenMotif replaces Lesstif

    We have replaced Lesstif with OpenMotif. All packages have been rebuilt accordingly, but there may be some problems. If you find any please report them to our bug tracker. Thanks!

  • FFMpeg upgraded

    FFMpeg was upgraded in -current this week. As with OpenMotif, all packages have been rebuilt accordingly, but there still may be some problems, in which case please report them to our bug tracker. Thanks!

  • New Pacman-G2 release

    We released a new pacman-g2 release today, which provides several fixes and improvements (including a fix for a segfault several people have encountered with the installer, fix will be in 0.7pre1), and a new -Qt option which helps to check for problems in your package database.

Tips and tricks

Here are some tips and tricks that have been discovered by our users to help make using your Frugalware system even easier and make your system even faster. Remember that this section relies on your contributions! If you have some tips and tricks that you would like to be shown in the newsletter, post them on the forums in the Tips and Tricks section!

  • Find out how much space is being used on your home directory

    If you find that your /home partition (or any other partition in fact) is low on disk space, the following command is useful:
    du -h --max-depth=1 <home directory>
    This will give you the total disk space being used by each sub-directory of the directory. Thanks to phayz for this tip.

  • Display extended search options in the GNOME search function

    Go to: /apps/gnome-search-tool/ in gconf-editor and enable: show_additional_options. Thanks to detto for this tip!

Security announcements

Here are a list of security issues which have been discovered and fixed in the 0.6 release in the past 2 weeks:

  • FSA183 - asterisk: Some vulnerabilities have been reported in Asterisk, which potentially can be exploited by malicious people to cause a DoS (Denial of Service) or compromise a vulnerable system. Please upgrade to asterisk-1.4.2-2terminus2.
  • FSA182 - mysql: Neil Kettle has reported a vulnerability in MySQL, which can be exploited by malicious users to cause a DoS (Denial of Service). Please upgrade to mysql-5.0.37-2terminus1.
  • FSA181 - php: Several vulnerabilities and weaknesses have been reported in PHP, where some have unknown impacts and others can be exploited by malicious users to manipulate certain data, disclose potentially sensitive information, bypass certain security restrictions, or to cause a DoS (Denial of Service), and potentially by malicious people to compromise a vulnerable system. Please upgrade to php-5.2.2-1terminus1.
  • FSA180 - xscreensaver: Alex Yamauchi has reported a weakness in XScreenSaver, which potentially can be exploited by malicious people to bypass certain security restrictions. Please upgrade to xscreensaver-5.01-2terminus1.
  • FSA179 - xmms: Secunia Research has discovered two vulnerabilities in XMMS, which can be exploited by malicious people to compromise a user's system. Please upgrade to xmms-1.2.10-12terminus1.
  • FSA178 - libexif: Victor Stinner has reported a vulnerability in libexif, which can be exploited by malicious people to cause a DoS and potentially compromise an application using the library. Please upgrade to libexif-0.6.13-2terminus1.
  • FSA177 - gimp: Marsu has discovered a vulnerability in Gimp, which can be exploited by malicious people to compromise a user's system. Please upgrade to gimp-2.2.13-2terminus1.
  • FSA176 - squirrelmail: Some vulnerabilities have been reported in SquirrelMail, which can be exploited by malicious people to conduct cross-site scripting and cross-site request forgery attacks. Please upgrade to squirrelmail-1.4.10-1terminus1.
  • FSA175 - python: Piotr Engelking has reported a security issue in Python, which can be exploited by malicious people to disclose potentially sensitive information. Please upgrade to python-2.5-3terminus1.
  • FSA174 - vim: Tomas Golembiovsky has discovered a vulnerability in Vim, which can be exploited by malicious people to compromise a user's system. Please upgrade to vim-7.0-4terminus1.
  • FSA173 - bind: A vulnerability has been reported in BIND, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to bind-9.4.1-1terminus1.
  • FSA172 - elinks: Arnaud Giersch has reported a weakness in ELinks, which potentially can be exploited by malicious, local users to gain escalated privileges. Please upgrade to elinks-0.11.2-2terminus1.
  • FSA171 - unzip: Race condition in Unzip 5.52 allows local users to modify permissions of arbitrary files via a hard link attack on a file while it is being decompressed, whose permissions are changed by Unzip after the decompression is complete. Please upgrade to unzip-5.52-2terminus1.
  • FSA170 - kernel: Two vulnerabilities and a security issue have been reported in the Linux Kernel, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to kernel-2.6.20-5terminus3.
  • FSA169 - truecrypt: When running without administrator privileges, TrueCrypt automatically attempts to elevate its access rights (if necessary) using the sudo command. Please upgrade to truecrypt-4.3-2terminus1.
  • FSA168 - postgresql: A security issue has been reported in PostgreSQL, which potentially can be exploited by malicious users to gain escalated privileges. Please upgrade to postgresql-8.2.4-1terminus1.
  • FSA167 - proftpd: A security issue has been reported in ProFTPD, which potentially can be exploited by malicious people to bypass certain security restrictions. Please upgrade to proftpd-1.3.0-7terminus1.
  • FSA166 - php: Ivan Fratric has reported a vulnerability in PHP, which potentially can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to php-5.2.1-3terminus1.
  • FSA165 - smb4k: Ben Hutchings discovered some security vulnerabilities in some utility programs included with smb4k. Please upgrade to smb4k-0.8.3-1terminus1.
  • FSA164 - ktorrent: Two vulnerabilities have been reported in KTorrent, which can be exploited by malicious people to overwrite arbitrary files on a user's system or to potentially compromise a user's system. Please upgrade to ktorrent-2.1.4-1terminus1.
  • FSA163 - lesstif: The XPM library's scan.c file may allow attackers to execute arbitrary code by crafting a malicious XPM image file containing a negative bitmap_unit value that provokes a buffer overflow. Please upgrade to lesstif-0.95.0-2terminus1.
  • FSA162 - phpmyadmin: Some vulnerabilities have been reported in phpMyAdmin, which can be exploited by malicious people to conduct cross-site scripting attacks. Please upgrade to phpmyadmin-2.10.1-1terminus1.
