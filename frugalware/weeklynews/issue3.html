News

There has been some interesting stuff happening in Frugalware. Here's a small overview of the best things:

  • Homepage design contest!

    We are running a contest to make a design for the next version of our homepage! I (Alex) decided that there's bound to be a better web designer out there than me, so therefore we are giving you the opportunity to put any web designing skills you may have to a good use, and help your favourite distro :)

    The rules are as follows:
    • No proprietary content such as Flash animations
    • The design must use CSS for layout and not tables
    • The design must use the official Frugalware logo
    • The design must be compatible with all browsers
    • Both the HTML and the CSS should pass W3C validation

    The closing date for entries is 1st June 2007. If you have an entry, you can either send it to the Frugalware-devel mailing list or visit IRC and poke AlexExtreme.

    The prize for the contest is eternal fame and glory from having credit given to you on the site, and a big bag of cookies from the developers :)

  • OpenOffice.org 2.2 in current

    The brand new OpenOffice.org 2.2 release was uploaded to current last week, with a new 'kerning' feature which improves the appearance of proportial fonts, and the ability to create bookmarks in PDF files.

  • Pacman-G2 changes

    The library behind pacman-g2 has been renamed to libpacman to make it more generic, as we hope that pacman-g2 will be suitable for use in any distro. Also, we have added some new C# bindings for libpacman, so that C# developers can use the awesome power of pacman-g2 in their apps :)

  • A new way to translate Frugalware to your language

    We now have a new web interface for translating our projects into your language! You can find out how to use begin translating with the new documentation page on translating.

  • Support for Xorg 6.x removed

    The symlinks to provide support for legacy apps which expect to find X under /usr/X11R6 have been removed. If you use any third-party apps which depend on this symlink, you should update them.

  • UTF-8 support for ncurses

    We have added the ncursesw libraries to the ncurses package to provide UTF-8 support.

  • Lots of work on the new GUI tools

    A lot of work has been done on some new GUI administration tools for 0.7. First up is GFpm, a replacement for the slow and outdated Frugalpkg. This one is written entirely in C using libpacman, making it much, much faster. Next is GNetConfig, a GUI for managing your network settings and creating network profiles. Finally, a new system tray icon is being created that'll allow for easy switching of network profiles. Of course, there are more tools to come, plus a control center shell to launch all the other tools.

Tips and tricks

Here are some tips and tricks that have been discovered by our users to help make using your Frugalware system even easier and make your system even faster:

  • Backup a list of packages installed

    Include the output of 'pacman-g2 -Qi' as part of your backup routine. If you find that a package sync/upgrade/removal breaks something you then have a record of the installed packages at a point in time. Again get the output of 'pacman-g2 -Qi', compare them (with the 'diff' tool) and you have somewhere to start in working out where things went wrong. Thanks to phayz for this tip!

  • Hiding specific desktop icons in XFCE.

    Some of you might not like to see Filesystem (mounted drives) icons or device icons or even the Home icon on your desktops. Heres how you can hide them:

    Create a file named 'xfdesktoprc' in /home/your_user_name/.config/xfce4/desktop and add the following lines to it:

    [file-icons]
    show-filesystem=false
    show-home=false
    show-removable=false

     After making the changes, make sure you logout and re-login into Xfce. Thanks to priyank for this tip!

  • Converting WMA files to MP3.

    To convert a .wma file to mp3 format, do this:
    (Note: you will need lame and mplayer installed.)

    $ mplayer -ao pcm:waveheader filename.wma
    $ lame -b 128 -h -m s -V 4 audiodump.wav

    You should then see a file called audiodump.wav.mp3. Just rename it to your filename.mp3

    You can change the bitrate by changing the value of -b option to lame.
    eg: for converting at a bitrate of 256kbps, use -b 256. Thanks to priyank for this tip!

Security announcements

Here are a list of security issues which have been discovered and fixed in the 0.6 release in the past 2 weeks:

  • FSA153 - qt4: Andreas Nolden has reported a vulnerability in Qt, which potentially can be exploited to conduct cross-site scripting attacks in applications using the Qt libraries. Please upgrade to qt4-4.2.3-2terminus1
  • FSA152 - madwifi: Some vulnerabilities have been reported in MadWifi, which can be exploited by malicious people to gain knowledge of potentially sensitive information or cause a DoS (Denial of Service). Please upgrade to madwifi-0.9.3-1terminus1
  • FSA151 - mod_perl: A vulnerability has been reported in mod_perl, which potentially can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to mod_perl-2.0.3-2terminus1
  • FSA150 - asterisk: A vulnerability has been reported in Asterisk, which can be exploited by malicious people to cause a DoS (Denial of Service). Please upgrade to asterisk-1.4.2-2terminus1
  • FSA149 - freetype2: A vulnerability has been reported in FreeType, which can be exploited by malicious people to cause a DoS (Denial of Service) and potentially compromise an application using the library. Please upgrade to freetype2-2.3.3-1terminus1
  • FSA148 - xorg-server: Some vulnerabilities have been reported in X.Org X11, which potentially can be exploited by malicious, local users to disclose sensitive information, cause a DoS (Denial of Service), and gain escalated privileges. Please upgrade to xorg-server-1.2.0-2terminus1
  • FSA147 - libxfont: Some vulnerabilities have been reported in X.Org X11, which potentially can be exploited by malicious, local users to disclose sensitive information, cause a DoS (Denial of Service), and gain escalated privileges. Please upgrade to libxfont-1.2.7-2terminus1
  • FSA146 - libx11: Some vulnerabilities have been reported in X.Org X11, which potentially can be exploited by malicious, local users to disclose sensitive information, cause a DoS (Denial of Service), and gain escalated privileges. Please upgrade to libx11-1.1.1-2terminus1
  • FSA145 - truecrypt: A security issue has been reported in TrueCrypt, which can be exploited by malicious, local users to cause a DoS (Denial of Service). Please upgrade to truecrypt-4.3-1terminus1
  • FSA144 - xine-lib: Some vulnerabilities have been reported in xine-lib, which can potentially be exploited by malicious people to compromise a vulnerable system. Please upgrade to xine-lib-1.1.4-3terminus1
  • FSA143 - evolution: Secunia Research has discovered a vulnerability in Evolution, which potentially can be exploited by malicious people to compromise a vulnerable system. Please upgrade to evolution-2.10.0-2terminus1
  • FSA142 - openoffice.org: Some vulnerabilities have been reported in OpenOffice.org, which potentially can be exploited by malicious people to compromise a user's system. Please upgrade to openoffice.org-2.1.0-6terminus1
