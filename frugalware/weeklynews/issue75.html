On 28 January 2011, VMiklos announced the availability of Frugalware 1.4 Release Candidate 2. As always, if you have the available time and bandwidth, please test this release and report any bugs you find. There will be one more release candidate before the final release of 1.4, so please help make 1.4 the best ever version.

Frugalware tweak! and Frugalware tweak 2! ???

If you read the frugalware-current commit log, or Bouleetbil's blog, you might have noticed that there is now a Frugalware tweak! and Frugalware tweak! 2. Please don't be alarmed, you're not going crazy. There really are two editions of Frugalware tweak!

FT2 is not a version upgrade of FT1 but instead the same application developed using a different language. Frugalware tweak! was first developed using Mono and, to please those who don't like to use Mono-based applications, Bouleetbil is re-developing it in Vala. The second "edition" is available in the 'current' repository but is in very early stages of its development. The first priority is to develop FT2 so it has the same functionality as FT1. Both editions will be maintained for at least the near future. If you haven't tried Frugalware tweak! you now have twice any many reasons to do so. :P

systemd init system will be available in Frugalware Linux 1.4

The Frugalware developers have successfully integrated the systemd init system. It will be available as an alternative to the current init system - sysvinit. Note that it will not be the default init system but will require 'activation' by the user.

systemd offers technical advantages over sysvinit but that's not the only reason it was chosen instead of other init systems. It's an unofficial Frugalware policy to work as part of the FLOSS community, for everyone's benefit. systemd was created by Lennart Poettering, the creator of PulseAudio, but people from other Linux distributions including OpenSUSE and Fedora are also part of its development. We benefit by sharing the maintenance and troubleshooting workload, also the experiences of everyone in the systemd user community.

Do-dec-a-he-what!?

A member of the Ubuntu community recently designed a dodecahedron which features a calendar for 2011, with each face featuring the dates of a month. Inspired by this, Frugalware artwork team member Centuri0 produced a Frugalware version of this calendar. I think the end result looks great! To create the calendar you print the design on a suitable piece of paper, then cut and fold as necessary to produce the final product. Go ahead and make it yourself, so that you have a constant reminder of your favourite Linux distribution. If you want to save money you could even give them to friends and family as gifts. The good part is that even if they throw them back at you, they won't hurt. :P

Tips and Tricks

Disclaimer - Be aware that the hints & tips provided here have NOT been tested and so come with no warranty.

Find your fastest mirror

This tip has appeared in the newsletter in the past but I thought it was worth repeating. When downloading packages, pacman-g2 requests the file from each of the mirrors listed in /etc/pacman-g2/repos/frugalware(-current), working through the list in order until successful or the file couldn't be found on any mirror. If the mirror(s) from which your packages are being downloaded is on the other side of the world, you're not likely to get the best download rate. To test this, use a tool such a wget to download a file from each of the mirrors that are closest to you and compare the download rates. If you find one which is faster than all the others, move it in the list of mirrors so that it is at the top.

Focus on Package(s)

When Frugalware developer ryuo asked me to review the following packages, I first thought of asking him for payment. He has recently added these to the 'current' repository so this seemed like free advertising. Now that I have had a chance to look at them I agree that they're very useful. I have decided, then, to include them without any fee being paid. Next time, though, the price will be double! :P

Both of the following applications were developed using the Qt framework, the basis of the KDE Software Collection. Even if you don't use KDE, you should take a look at these two because they depend only on Qt, not KDE itself. They would fit in just about anywhere and provide some very useful functionality.

Juffed

Juffed is an advanced text editor and in addition to the standard features of any text editor, it offers:

  • syntax highlighting;
  • find in files;
  • autocompletion;
  • code folding.

It offers two useful features which help editing on commonly-used files: favourites and sessions. The favourites feature allows you to maintain a list of files which you use commonly. This might include files containing snippets of code or your projects To Do list, for example. The sessions feature is common to other editors and allows you to easily work on a group of files at a time. You can save a session with a meaningful name and switch between sessions as you need.

Qedit

Like Juffed, qedit is an advanced text editor developed using the Qt framework. It has a slightly different set of features than Juffed, including:

  • syntax highlighting;
  • spell checking;
  • code folding.

Qedit offers a feature named "multiple views". It allows you to split the current window into multiple panes, containing another portion of the same file or a different file. This can be very useful when you're working on related pieces of code in the same file, or particularly when they are in separate files.

To provide some customisation of Qedit's behaviour when editing files of different types, it offers a featurre named "Document classes". Each document class is a group of settings applied to all files where either the name or first line matches a regular expression. The settings can be easily modified via a GUI interface and include the following:

  • syntax highlighting patterns;
  • indentation rules;
  • text edition macros;
  • matching parenthesis highlighting;
  • logical blocks markers.

Try Qedit if you're looking for an advanced text editor which is fast, friendly and allows you to see different parts of the same file in one view.
