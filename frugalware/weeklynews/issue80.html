Orphaned packages find new "homes"

A Frugalware package whose maintainer is no longer active is considered "orphaned", since there's no-one caring for it any more. A list of all orphaned packages is maintained on a wiki page and developers are encouraged to take over their ownership. Recently many of them have been adopted by new parents (maintainers) and the list has been gradually getting shorter. When a package is adopted everyone has tears in their eyes, although for the new maintainer I'm not sure if they're tears of joy. It's not certain what happens if we can't find a maintainer for an orphaned package, although there's always /dev/null.

Tips and Tricks

New features for pacman-g2

Two new features were recently added to pacman-g2: a new operation and a new Query option.

pacman-g2 -P

The new operation uses the parameter "-P" or "--ps" and lists all processes which are using deleted files. From this information you can confirm if any services are affected and if so, restart them.

pacman-g2 -Qf

The new Query option "-Qf" checks the specified package or all packages (if a specific package is not given) to confirm if all the files owned by that package are present. This option is very useful if you believe a package might be incomplete; perhaps you think a library file might be missing. Here's sample output you would see if a file was missing:

pacman-g2 /usr/share/doc/pacman-g2-3.8.2/TODO	No such file or directory.

For full details of all pacman-g2's operations and options, refer to its man page.

Maintain a list of installed packages

This tip has featured in the newsletter before but it's definitely worth repeating. Most of us install the same packages, no matter which PC we're working on, but it's impossible to remember all their names. If you run the command "pacman-g2 -Q" (with root privileges) and redirect its output to a file, you have a record of all installed packages. If you want to have this list automatically updated, set it up as a cron job, perhaps using a file name based on the current date.

To take this tip one step further, the output of the above command can be used as input to pacman-g2 to install all listed packages. This can be very useful if for example, you're installing on another computer and want exactly the same set of packages or have deleted a Frugalware installation and are starting again. Using this method you only need to install the 'base' group of packages then use the following command to install everything listed in the file.

pacman-g2 -S --ask 32 `sed 's/ .*//' file`

Thanks to VMiklos for providing the command used in this tip.

Focus on Packages

Minitube

Watching YouTube videos in a web browser has its problems, since it requires a Flash plugin. Minitube aims to make this easier because it allows you to watch them in a GUI application instead. The user interface is quite simple, with a search input field that allows you to search videos or channels by keywords. Once you have found what you're looking for you can play the video immediately or add it to a playlist. Often people publish to YouTube educational videos or "screencasts" and Minitube's playlist feature is very useful because you can set up the videos to play in order.

Playing a video is very easy to operate, with the standard controls in a bar just below the main menu. You can choose from three user interface modes while playing a video: standard, compact and fullscreen. If you want to save the video to watch later, you simply choose Video > Download from the main menu. If instead you want to refer to the video, perhaps to link to it in a blog post or email, you can copy the URL for the YouTube page or the YouTube stream alone, again from the Video menu.

Minitube is perhaps the easiest way of finding and playing YouTube videos. It's definitely recommended for all Frugalware users who have visited YouTube more than once.

Plank

There are some applications of which developers can't seem to resist writing their own version. The FLOSS world has more text editors and media players than most people have had hot meals. The latest trend seems to be docks, those GUI panels which appear at the side of the screen and contain launchers and various widgets. Plank is YAD (Yet Another Dock) being developed in Vala by the elementaryOS core team. It's a very new project and doesn't have all the features of other docks but work is progressing quickly. Plank has very few dependencies so it's suitable for use with a desktop environment or standalone window manager.

Like most other docks it shows a set of icons which when clicked launch the matching application. A light glow appears under an icon if that application is already running. If you click on the icon of a running application, Plank brings that application into focus and if needed, switches desktops. To keep out of your way, Plank can be set to always hide, requiring you to move the mouse cursor near it before it will appear, or hide intelligently, in whch case it will hide only when overlapped by other windows. Try Plank for yourself because after all, one more dock isn't going to hurt anyone. :)

From the wiki to you...

The wiki is a very useful source of information about Frugalware, hardware and applications. Sometimes the question is asked: what should go in the wiki and what should go in the official manual? The general guideline is that the manual's purpose is to help you install, manage and maintain Frugalware only, not individual applications. The wiki's purpose is to allow everyone to contribute any information that they think might be useful to other Frugalware users. Changes occur so quickly in the FLOSS world that it would impossible to cover everything in the official documentation but a wiki allows collective knowledge to be shared, which is itself a great feature of the FLOSS community.
