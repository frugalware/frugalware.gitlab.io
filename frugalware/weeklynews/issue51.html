Vanilla packages

Frugalware's packages are as close as possible to the source code provided by the project. Frugalware-specific patches are only applied when necessary - e.g. to fix a security bug for which an official release hasn't been made. There are several good reasons for using "vanilla" (unpatched) source code, including:

  • patches are the distribution's responsibility and we don't need any extra burden;
  • patches may introduce bugs or security issues;
  • support is available directly from the project.

RTFM

Anyone who has asked a technical question has probably been given the following answer at some time: RTFM, which is an abbreviation for Read The Fine Manual. Sometimes the word "Fine" is replaced with another word which also starts with the letter 'F' but I can't remember which word. :P Even experienced Linux users should take this advice and read the documentation which was provided with the applications that they use. Of course you might think that you can already do all that you need. However the documentation might describe a more efficient method, possibly tell you about great features that are not obvious. When you first install a package, please take the time to read its documentation, even if it's only a README. You just might learn something new!

Transparency versus performance

Several window managers offer real transparency, in which the contents of any windows below another window are visible, usually with some opaqueness. To provide this feature requires a lot of computation by the CPU and/or graphics card. Providing this feature while windows are moved around the desktop is an even greater load, and some PCs simply can't keep up. Your PC is probably too slow if the movement of windows lags behind the movement of the mouse's cursor. To reduce the workload, you can turn off the display of windows's contents while they're being moved. Gnome's default window manager - Metacity - offers an option in GConf named "reduced resources", which turns off some of the "eye candy" effects, with a performance improvement. Read your window manager's documentation to confirm if it offers a similar option, and how to turn it on (and off).

Focus On Package(s)

Code Browser

Code Browser is a text editor whose main purpose is editing source code. It supports folding, a feature of many editors that allows sections of code to be folded and restored. Folding of a section of code results in only the first part of the section being visible, allowing you to focus more closely on specific sections. Code Browser has an option that I've not seen in an editor before - it can display the code in a tree view. Working on a section of code requires you to navigate the tree and click on the node which contains the source code. The tree view is useful because it gives illustrates the overall structure of your code. What's really interesting about Code Browser is that it does this without changing the structure of the text or code you're working on. It simply adds special markers in the form of comments. Please look at the screenshot below and, if you're interested, look at Code Browser's home page. For those that love both Windows and Linux, Code Browser is available for both platforms.
